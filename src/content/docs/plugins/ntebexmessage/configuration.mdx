NTebexMessage uses two configuration files.

| File | Purpose |
| --- | --- |
| `config.yml` | Main configuration: custom arguments, templates, buttons and reward event. |
| `discord.yml` | Discord webhook configuration: bot identity, content message and embed. |

After editing either file, reload the plugin:

```
/tebexmessage reload
```

## Top-level structure

```yaml
customs-args: []        # custom argument names

separator-symbol: "&7•"  # joins multiple packages

broadcast:
  templates:
    one:        { … }
    multiple:   { … }
    # any number of additional templates

buttons:
  center: 20
  list:
    shop:     { … }
    discord:  { … }

reward-event:
  words: []
  players-write-word: 5
  ignore-case: true
  cooldown:    { enabled, time }
  players-rewards-commands: []
  final-message: { enabled, message }
```

## Permissions

| Permission | Description | Default |
| --- | --- | --- |
| `tebexmessage.help` | Display the help menu. | everyone |
| `tebexmessage.reload` | Reload the configuration. | OP |
| `tebexmessage.args` | List custom arguments. | OP |
| `tebexmessage.admin` | Bypasses every permission check. | OP |
