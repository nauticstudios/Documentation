NTebexMessage ships with three configuration files. All of them live under `/plugins/NTebexMessage/`.

| File | Purpose |
| --- | --- |
| `config.yml` | Main configuration — custom arguments, templates, buttons and reward event. |
| `messages.yml` | UI strings used by `/ntb help`, `/ntb args`, error messages and console output. |
| `discord.yml` | Discord webhook configuration — bot identity, filters and the active mode (classic or Components V2). |

After editing any of them, reload the plugin:

```
/ntb reload
```

## `config.yml` — top-level structure

```yaml title="config.yml"
customs-args: []        # custom argument names

separator-symbol: "&7•" # joins multiple packages

broadcast:
  templates:
    one:        { … }
    multiple:   { … }
    # any number of additional templates

buttons:
  center: 20
  list:
    shop:     { … }
    discord:  { … }

reward-event:
  words: []
  players-write-word: 30
  ignore-case: true
  cooldown:                { enabled, time }
  bossbar:                 { enabled, text, color, style }
  actionbar:               { enabled, text }
  null-players: "..."
  players-rewards-commands: []
  final-message:           { enabled, message }
```

## `messages.yml` — UI strings

Customise every player-facing and console-facing string.

```yaml title="messages.yml"
help: [ ... ]            # /ntb help output
args:                    # /ntb args output
  header: [ ... ]
  empty: "..."
  format: "..."
  footer: [ ... ]

console:
  usage: [ ... ]
  invalid-format: "..."
  unknown-arg: "..."
  available-args: "..."
  missing-parenthesis: "..."

reload: "..."
no-permission: "..."
missing-arg: "..."
console-only: "..."
```

Placeholders supported in these strings:

| Placeholder | Replaces with |
| --- | --- |
| `{command}` | The label the player used (`ntb`, `tb`, `tebexmessage`, ...). |
| `{arg}` | The offending argument name (in error messages). |
| `{args}` | Comma-separated list of available custom arguments. |
| `{usage}` | Auto-built usage line for the console. |

## `discord.yml` — Discord webhook

Full reference and per-mode examples on the [Discord webhook](/plugins/ntebexmessage/webhook/) page.

```yaml title="discord.yml"
webhook:
  enabled: false
  url: "INSERT-WEBHOOK-URL"
  username: "Nautic Studios"
  avatar: "https://i.imgur.com/ip1xV13.png"
  mode: components-v2   # or: classic

  filters:              { strip-words, replace, multi-package }
  classic:              { content, embed, buttons }
  components-v2:        { accent, layout }
```

## Permissions

| Permission | Description | Default |
| --- | --- | --- |
| `tebexmessage.help` | Display `/ntb help`. | OP |
| `tebexmessage.reload` | Reload all configuration. | OP |
| `tebexmessage.args` | List custom arguments. | OP |
| `tebexmessage.admin` | Bypasses every permission check. | OP |
