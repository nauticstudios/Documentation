**NTebexMessage** turns Tebex store purchases into rich in-game broadcasts. Each purchase can trigger a chat message, title, actionbar, bossbar, sound, interactive buttons, a reward event and a Discord webhook — all driven from configuration files.

## At a glance

<CardGrid>
  <Card title="Templates" icon="document">
    Multiple broadcast layouts. The console picks one by name when announcing a purchase.
  </Card>
  <Card title="Interactive buttons" icon="cursor">
    Clickable chat components that open store, Discord or any URL.
  </Card>
  <Card title="Reward event" icon="rocket">
    Players type a configured word in chat to receive automatic rewards.
  </Card>
  <Card title="Discord webhook" icon="discord">
    Send embed-rich notifications to a Discord channel on every purchase.
  </Card>
</CardGrid>

## Quick example

A purchase is announced from console:

```
/tebexmessage one player(Senkex) package(VIP) price($10)
```

This runs the `one` template defined in `config.yml`, replaces `{player}`, `{package}` and `{price}`, and broadcasts everything the template has enabled — chat lines, title, sound, actionbar, bossbar, buttons and any optional event.

## How it fits together

<Steps>
1. The Tebex plugin or a CSP runs `/tebexmessage <template> arg(value) …` from console after a purchase.

2. NTebexMessage looks up the template by name in `config.yml`.

3. Custom arguments are replaced into placeholders (`{player}`, `{package}`, `{price}` …).

4. The configured chat message, titles, sounds and bars are dispatched to players.

5. Optional events fire: reward event, Discord webhook, player head display.
</Steps>
