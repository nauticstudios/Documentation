AtlasLang is the multilingual engine used internally by every Nautic Studios plugin. It stores each player's language in a database, loads YAML message files per language, and exposes a strict PlaceholderAPI integration and a public developer API.

GitHub: [github.com/nauticstudios/atlaslang](https://github.com/nauticstudios/atlaslang)

## Features

- Per-player language stored in the database
- Recursive YAML loading per language folder
- Strict language validation with safe fallbacks
- PlaceholderAPI expansion
- Public developer API
- Optional GitHub synchronization

## Configuration

Languages are declared in the main configuration file. The format is `locale:folder:defaultFile`.

```yaml
default: english

register:
  languages:
    - en:english:messages.yml
    - es:spanish:messages.yml
```

Each entry must point to an existing folder under `/plugins/AtlasLang/languages/`. The default file is created automatically if it does not exist.

Folder layout:

```
languages/
 ├── english/
 │   ├── messages.yml
 │   └── scoreboard.yml
 └── spanish/
     ├── messages.yml
     └── scoreboard.yml
```

:::caution
Malformed or missing language files will never crash the server. A safe fallback message is returned instead.
:::

## Placeholders

AtlasLang registers the `alang` PlaceholderAPI identifier.

```
%alang_(file)_(path)%
```

Examples:

```
%alang_(messages)_(join_message)%
%alang_(lobby:scoreboard)_(title)%
```

Resolution always uses the language stored for the player. If the player has no language set, the default is used. Invalid file or path keys return a safe error string.

:::note
Placeholders that do not follow the `(file)_(path)` format will return an invalid-format message.
:::

## Commands

Base command: `/atlaslang`

| Command | Permission |
| --- | --- |
| `/atlaslang help` | — |
| `/atlaslang list` | `atlaslang.list` |
| `/atlaslang aliases` | `atlaslang.aliases` |
| `/atlaslang set <player> <language>` | `atlaslang.set` |
| `/atlaslang reset <player>` | `atlaslang.reset` |
| `/atlaslang info <player>` | `atlaslang.info` |
| `/atlaslang reload` | `atlaslang.reload` |
| `/atlaslang github sync` | `atlaslang.github` |
| `/atlaslang github status` | `atlaslang.github` |

Use `atlaslang.admin` to grant access to every command.

## Developer API

The public API is accessed through static methods on `AtlasAPI`.

```java
AtlasAPI.get(player, "path.to.message");
AtlasAPI.getOrDefault(player, "path.to.message", "Fallback");
AtlasAPI.setLanguage(player, "english");
AtlasAPI.getLanguage(player);
AtlasAPI.has("english", "path.to.message");
AtlasAPI.isLanguageRegistered("english");
```

Source: [AtlasAPI.java](https://github.com/nauticstudios/AtlasLang/blob/master/src/main/java/com/github/nautic/api/AtlasAPI.java)

## GitHub synchronization

Language files can be synced from a remote repository at runtime. Sync runs asynchronously and never blocks the main thread.

```yaml
github:
  repository:
    name: "owner/repository"
    branch: "main"

  authentication:
    type: "none"
    token: ""

  paths:
    remote-root: "languages"
    local-root: "languages"

  sync:
    create-missing: true
    overwrite-existing: true
    delete-missing: false
    reload-after-sync: true
```

Optional token authentication is supported for private repositories.
