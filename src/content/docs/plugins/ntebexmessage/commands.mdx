## Command reference

| Command | Permission | Default | Description |
| --- | --- | --- | --- |
| `/tebexmessage help` | `tebexmessage.help` | everyone | Display the help menu. |
| `/tebexmessage reload` | `tebexmessage.reload` | OP | Reload all configuration files. |
| `/tebexmessage args` | `tebexmessage.args` | OP | List all configured custom arguments. |
| `/tebexmessage <template> arg(value)…` | console only | console | Broadcast a purchase using the given template. |

Use `tebexmessage.admin` to grant access to every command.

## Broadcast command

The broadcast command runs only from console.

```
/tebexmessage <template> arg(value) arg(value) …
```

- `<template>` — name of a template defined in `config.yml`.
- `arg(value)` — one custom argument per pair, in any order.

### Example

```
/tebexmessage one player(Senkex) package(VIP) price($10)
```

| Argument | Value |
| --- | --- |
| Template | `one` |
| `{player}` | `Senkex` |
| `{package}` | `VIP` |
| `{price}` | `$10` |

The template's chat message, title, sound, actionbar, bossbar, buttons and any enabled event will be triggered.

## Custom arguments

Arguments are declared in `config.yml`:

```yaml
customs-args:
  - player
  - package
  - price
```

Every argument listed here becomes a placeholder that can be used inside templates, messages, titles, actionbars, bossbars and Discord webhooks.

```
{player}
{package}
{price}
```

:::caution
If an argument used in the command is not declared in `customs-args`, the command returns an error.
:::
