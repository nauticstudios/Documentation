**NTebexMessage** turns Tebex store purchases into rich in-game broadcasts. Each purchase can trigger a chat message, title, actionbar, bossbar, sound, interactive buttons, the buyer's head, a chat reward event and a Discord webhook — all driven from configuration files, with hot reload and Folia support.

| | |
| --- | --- |
| **Base command** | `/ntb` |
| **Aliases** | `ntebex`, `ntebexmessage`, `tb`, `tbmsg`, `tbxm`, `tbxmsg`, `tebexmessage` |
| **Folia** | Supported |
| **Min. API** | Bukkit `1.16` |
| **Soft-depend** | PlaceholderAPI |

## Quick example

A purchase is announced from console:

```
/ntb one player(Senkex) package(VIP) price($10)
```

This runs the `one` template defined in `config.yml`, replaces `{player}`, `{package}` and `{price}`, and broadcasts everything the template has enabled — chat lines, title, sound, actionbar, bossbar, buttons and any optional event.

## How it fits together

<Steps>
1. The Tebex plugin (or a CSP) runs `/ntb <template> arg(value) …` from console after a purchase.

2. NTebexMessage looks up the template by name in `config.yml`.

3. Custom arguments are replaced into placeholders (`{player}`, `{package}`, `{price}` and any custom keys you declare).

4. The configured chat message, titles, sounds and bars are dispatched to players.

5. Optional events fire: reward event, Discord webhook, buyer's head display.
</Steps>
