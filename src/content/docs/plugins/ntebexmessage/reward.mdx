The reward event lets players participate in a short chat challenge after a broadcast. The first players to type one of the configured words receive an automatic reward executed by console commands.

## How it works

<Steps>
1. A template with `events.reward-event: true` is broadcasted.

2. The reward event starts and listens for chat messages.

3. Players type any of the configured words. Each unique player counts once.

4. When the player limit is reached or the cooldown expires, the event ends.

5. The configured reward commands run for every player who participated, and a final message is broadcasted.
</Steps>

## Enable the event

In the template that should trigger it:

```yaml
events:
  reward-event: true
```

## Configured words

Any of these words counts as a valid entry.

```yaml
reward-event:
  words:
    - gg
    - hype
```

## Player limit

The maximum number of players that can be rewarded. The event ends automatically when this number is reached.

```yaml
reward-event:
  players-write-word: 5
```

## Case sensitivity

```yaml
reward-event:
  ignore-case: true
```

When `true`, `gg`, `GG` and `Gg` all count as the same word.

## Timeout

The cooldown defines how long the event stays open before automatically ending if the player limit is not reached.

```yaml
reward-event:
  cooldown:
    enabled: true
    time: 15
```

`time` is in seconds.

## Reward commands

Commands executed once per participating player. `%player%` is replaced with the player's name.

```yaml
reward-event:
  players-rewards-commands:
    - "give %player% diamond 1"
    - "eco give %player% 500"
```

## Final message

Broadcasted when the event ends. Supports `<center>` and `<empty>` tags.

```yaml
reward-event:
  final-message:
    enabled: true
    message:
      - "<center>&aReward event finished!</center>"
      - "%players_reward_claimed%"
```

| Placeholder | Replaces with |
| --- | --- |
| `%players_reward_claimed%` | The list of players who received rewards. |
